【又6】QT交叉编译

发表于 2013-01-01 | 分类于 个人生活

1. 安装linux arm gcc编译工具：

从网上下载arm-linux-gcc编译工具包，将工具包文件夹拷贝到Linux PC；将安装包复制到usr/local目录下；修改环境变量，保存当前的usr/local为系统环境变量目录

在Termination中输入arm-linux-gcc –v，如果可以输出arm linux版本信息，代表安装成功

 

2. 编译QT4 lib库

从http://qt-project.org/downloads上下载QT source code qt-everywhere-opensource-src-4.6.0.tar.gz：

并拷贝到Linux PC，解压缩到某个目录下，文件夹以qt命名。

在Termination中利用cd命令将当前路径切换到qt目录，输入ls，检查qt目录下是否有config程序，然后输入

./configure

-prefix /usr/local/QT_Emb/QtEmbedded-4.6.0-arm

-opensource

-confirm-license

-release -shared

-embedded arm

-xplatform qws/linux-arm-g++

-depths 16,18,24

-fast

-optimized-qmake

-pch

-qt-sql-sqlite

-qt-libjpeg

-qt-zlib

-qt-libpng

-qt-freetype

-little-endian -host-little-endian

-no-qt3support

-no-libtiff -no-libmng

-no-opengl

-no-mmx -no-sse -no-sse2

-no-3dnow

-no-openssl

-no-webkit

-no-qvfb

-no-phonon

-no-nis

-no-opengl

-no-cups

-no-glib

-no-xcursor -no-xfixes -no-xrandr -no-xrender

-no-separate-debug-info

-nomake examples -nomake tools -nomake docs

 

执行成功后，输入

 

./make

./make install

 

全部执行完后，会生成/usr/local/Trolltech/QT_Emb/QtEmbedded-4.6.0-arm，这个目录下包含了所以QT的lib，和QT creator需要用到的qmake等文件，需要提醒的是，整个编译过程时间较长，特别是make

 

3. 安装设置QT Creator和编译执行

通过软件中心搜索带QT4的软件，并安装QTCreator可能需要通过在termination输入sudo apt-get install qtcreator安装。打开QTCreator, 在菜单Tools—>Options打开Options窗口，在左侧树形控件中选择Qt4—>Qt Versions，在右侧界面点“+”，加入新的QT settings，命名为QTARM, Qmake程序的路径，选择指向/usr/local/QT_Emb/QtEmbedded-4.6.0-arm/下你make的qmake的路劲，然后点击Rebuild按钮，等待片刻，看到Debugging Helper，后出现一个绿色的“√”即可。

在编译代码时，默认情况下是PC版本，如果要用QTCreator编译成开发板上可运行的程序，需要在project选项选择对应的程序，在Build settings部分点Add，添加然后在开始编译前将Build configuration选择成QTARM，然后Build。
